## Troubleshooting

### Issue: "Deploy" button is grayed out

**Solution:** Make sure you've selected a business first. Click on a business name in the list, then try again.

### Issue: Deployment stuck at "Creating GitHub Repository"

**Solution:**
1. Wait 1-2 minutes (GitHub can be slow sometimes)
2. If still stuck after 2 minutes, refresh the page
3. Check your internet connection
4. Try again with "Manual Deploy" option

### Issue: Can't see "Publish to GitHub" in Lovable

**Solution:**
1. Make sure you're logged into Lovable
2. Look for a "Publish" or "Export" button in the top right
3. Contact technical support if you can't find it

### Issue: Deployment failed with error message

**Solution:**
1. Read the error message carefully
2. Try clicking "Deploy" button again (automatic retry)
3. If error persists, use "Manual Deploy" option instead
4. Contact technical support with the error message

### Issue: Customer wants changes to their website

**Solution:**
1. Click the "Lovable URL" link
2. Make the requested changes in Lovable
3. Click "Publish to GitHub" again
4. System will automatically update the live website
5. Changes appear in 2-3 minutes

## Frequently Asked Questions

### Q: How long does deployment take?

**A:** About 3-5 minutes with automated deploy. The first deployment takes longest because Render needs to build the website. Subsequent updates are faster.

### Q: Can I make changes after deployment?

**A:** Yes! Click the "Lovable URL" link, make changes, and click "Publish to GitHub" again. The system will automatically update the live website.

### Q: What if the customer doesn't like the AI-generated website?

**A:** You can make unlimited edits in Lovable before publishing. The AI does a great job, but you have full control to customize everything.

### Q: Do I need to know GitHub or coding?

**A:** No! The system handles all the technical parts automatically. You just need to click buttons and share the final website link.

### Q: What if automated deploy fails?

**A:** Use the "Manual Deploy" option as a fallback. It requires a few extra steps (copying and pasting the GitHub URL), but still works great.

### Q: Can I deploy multiple websites for the same business?

**A:** Yes, but each deployment creates a new website. If the customer wants to replace their existing website, deploy again with the same package.

### Q: How do I handle refunds?

**A:** Contact your manager or technical support. The dashboard doesn't handle refunds - you'll need to process them through your payment processor.

### Q: What happens if Render deployment fails?

**A:** The system will automatically retry up to 3 times. If it still fails, you'll see an error message with instructions. Contact technical support for help.
